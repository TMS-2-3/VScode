調子・攻防メーター仕様メモ
更新日: 2026-07-02
基準: 現在のコード実装(data/balance.js, battle/stats.js, battle/combat.js, battle/ai.js, battle/skills.js)

===============================================================================
1. 対象
===============================================================================

調子を持つ対象:
- 味方CPのみ。
- 現在はウルペス、リハス、スシア。
- アルジュナ/Finaldは調子を持たない。
- 敵は調子を持たない。

攻防メーターを持つ対象:
- 味方CPのみ。
- アルジュナ/Finaldは攻防メーターを持たない。
- 敵は攻防メーターを持たない。

初期値:
- 調子: 35
- 攻防メーター: 0
- 実適用攻防メーター: 0

範囲:
- 調子: 0から100
- 攻防メーター: -2から+2の5段階

===============================================================================
2. 調子の基本
===============================================================================

調子は0から100の値。
高いほど攻撃性能や移動が強くなるが、被ダメージも増える。
低いほど与ダメージが下がり、必殺技も使えなくなる。

調子50を基準値として扱う処理がある。
現在は移動速度補正で使っている。

味方CPの必殺技:
- 調子50以下だと手動/自動ともに使用不可。
- 使用しようとすると「不調」と表示される。

自動必殺:
- 調子95以上、必殺ゲージ満タン、行動可能なら即発動する。
- 調子85以上、必殺ゲージ満タン、行動可能なら行動判定時に16%で発動する。
- 敵予兆回避中や行動不能中は発動しない。

===============================================================================
3. 調子の自然変動
===============================================================================

毎秒、現在HP割合に応じて調子が変動する。
対象は味方CPのみ。

HP割合:
- 85%以上: +0.5 / 秒
- 70%以上: +0.2 / 秒
- 60%以上: +0.0 / 秒
- 50%以上: -0.5 / 秒
- 40%以上: -0.8 / 秒
- 30%以上: -1.0 / 秒
- 20%以上: -1.5 / 秒
- 10%以上: -3.0 / 秒
- 10%未満: -5.0 / 秒

この自然変動にも攻防メーターの調子増減倍率がかかる。
- 増加なら moodGain 倍率。
- 減少なら moodLoss 倍率。

調子90以上の増加減衰:
- 調子が90以上の時、調子増加量は0.5倍。
- 90未満から90をまたぐ増加の場合、90を超えた分だけ0.5倍。
- 減少にはこの減衰はかからない。

===============================================================================
4. 調子の増加共通処理
===============================================================================

調子増加は基本的に addMoodGain を通る。

増加量にかかる補正順:
1. 無被弾時間ボーナス
2. 攻防メーターの moodGain 倍率
3. 調子90以上の増加減衰
4. 0から100に丸める

無被弾時間ボーナス:
- noDamage が3秒以下: x1.0
- noDamage が3秒超えから18秒まで: x1.0からx1.3へ線形増加
- noDamage が18秒以上: x1.3

noDamage:
- 戦闘中、毎秒加算される。
- HPダメージを受けると0に戻る。
- シールドで完全に防いだ分はHPダメージではないため、基本的に被ダメ扱いにならない。

1行動あたりの調子増加上限:
- 敵への攻撃、撃破、複数ヒットボーナスなど、行動由来の増加は addMoodActionGain を通る。
- 1行動あたり最大+6。
- 行動開始時にその行動の累積増加量がリセットされる。

===============================================================================
5. 敵にダメージを与えた時の調子増加
===============================================================================

味方CPが敵にダメージを与えた時に増加する。
アルジュナ/Finaldは調子を持たないので対象外。

基礎計算:
- 敵最大HPに対する実ダメージ割合を使う。
- gain = 実ダメージ割合 * 12 * 距離補正
- 1ヒット上限は+2。
- 1行動上限は+6。

実ダメージ割合:
- min(敵の被弾前HP, 報酬対象ダメージ) / 敵最大HP
- リハスの痛みなしで遅延に回した分も、攻撃者側の調子増加計算には含まれる。
- シールド吸収分はこの調子増加計算には基本的に含まれない。

距離補正:
- 200px以下: x1.0
- 200pxから420px: x1.0からx1.5へ線形増加
- 420px以上: x1.5

敵撃破ボーナス:
- 敵を倒すと+3。
- これも1行動上限+6の中に含まれる。

複数ヒットボーナス:
- 1行動で敵を2体以上巻き込むと発生。
- bonus = 3 + ヒットした敵数
- これも1行動上限+6の中に含まれる。

===============================================================================
6. 被ダメージ時の調子減少
===============================================================================

味方CPがHPダメージを受けた時に減少する。
アルジュナ/Finaldは対象外。

基礎計算:
- 実際にHPが減った量だけを見る。
- シールドで防いだ分はHPが減っていないため、調子減少に含まれない。
- 痛みなしで遅延された分は、遅延ダメージの各tick時に別途減少する。

計算式:
- damageRatio = 実HPダメージ / 現在の最大HP
- referenceDamage = damageRatio * 現在の最大HP
- つまり現状は referenceDamage = 実HPダメージ と同じ。
- loss = referenceDamage * 0.0495 * 1.2 + 0.2
- その後、攻防メーターの moodLoss 倍率がかかる。

注意:
- 被ダメージによる調子減少は、固定の調子参照HPではなく現在の最大HPを使う。
- 装備やセット効果などで最大HPが変わっている場合、その最大HPを基準にする。

基礎減少:
- 被ダメージ時は最低でも基礎値0.2が減少計算に入る。
- この0.2にも攻防メーターの moodLoss 倍率がかかる。

===============================================================================
7. 回復・シールドによる調子増加
===============================================================================

ヒールでHPが回復した時:
- 対象が味方CPなら調子が増える。
- noMoodオプションがある場合は増えない。
- 自分への回復など、一部の処理ではnoMoodが付く。

ヒール増加計算:
- healRatio = 実回復量 / 最大HP
- referenceHeal = healRatio * 調子参照HP
- gain = referenceHeal * 0.1104 * 1.2 + クイックヒールボーナス
- その後、無被弾時間ボーナス、攻防moodGain、90以上減衰がかかる。

クイックヒールボーナス:
- 対象のnoDamageが3秒未満なら+2。
- つまり、被弾直後の回復は調子が上がりやすい。

シールド付与:
- シールドが実際に付与された場合、支援必殺ゲージ獲得対象になる。
- シールドが実際に付与された量に応じて、対象の調子が即時増加する。
- shieldRatio = 実付与シールド量 / 現在の最大HP
- referenceShield = shieldRatio * 現在の最大HP
- つまり現状は referenceShield = 実付与シールド量 と同じ。
- gain = referenceShield * 0.1104 * 1.2
- 回復と違い、被ダメ直後のクイックヒールボーナス+2は付かない。
- シェルト固有の固定調子増加量は持たない。
- その後、無被弾時間ボーナス、攻防moodGain、90以上減衰がかかる。

シールド維持中:
- シールド耐久が残っている間、毎秒+0.3。
- これも addMoodGain を通るため、無被弾時間、攻防moodGain、90以上減衰の影響を受ける。

===============================================================================
8. 調子による戦闘補正
===============================================================================

与ダメージ倍率:
- 調子10以下: x0.3
- 調子10から50: x0.3からx1.0へ線形増加
- 調子50から90: x1.0からx1.3へ線形増加
- 調子90以上: x1.3

被ダメージ倍率:
- 調子70以下: x1.0
- 調子70から80: x1.0からx1.1へ線形増加
- 調子80から90: x1.1からx1.25へ線形増加
- 調子90から100: x1.25からx1.4へ線形増加

注意:
- 高調子は与ダメージが上がるが、被ダメージも増える。
- 低調子は与ダメージが大きく落ちる。

移動速度補正:
- 調子50を超えた分だけ移動速度が上がる。
- 倍率 = 1 + max(0, 調子 - 50) * 0.003
- 調子100なら x1.15。
- 通常移動と予兆回避の移動に使われる。

味方巻き込み:
- 味方への攻撃判定は、攻撃者の調子が80未満なら基本的に当たらない。
- 調子80以上90未満: 味方にも当たり、ダメージx0.3、効果時間x0.5。
- 調子90以上: 味方にも当たり、ダメージx0.8、効果時間x1.0。
- 強制敵対ターゲットなどの特殊状態ではこの制限を無視する。

===============================================================================
9. 調子による適正距離補正
===============================================================================

攻防メーターで決まる基本適正距離に、調子補正がかかる。

高調子:
- 調子75以上で近づく。
- 調子75から100で距離倍率がx1.0からx0.6へ線形変化。

低調子:
- 調子30以下で離れる。
- 調子30から0で距離倍率がx1.0からx1.4へ線形変化。

例:
- 攻防0の前衛距離65px、調子100なら 65 * 0.6 = 39px。
- 攻防0の前衛距離65px、調子0なら 65 * 1.4 = 91px。

===============================================================================
10. 攻防メーターの基本
===============================================================================

攻防メーターは -2, -1, 0, +1, +2 の5段階。

意味:
- -2: 防御大
- -1: 防御
-  0: 中立
- +1: 攻撃
- +2: 攻撃大

内部値:
- commandBias: 表示値、次に反映される予定値。
- activeCommandBias: 現在実際に効いている値。

反映タイミング:
- 指示で commandBias はすぐ変わる。
- activeCommandBias は、そのCPが次の行動を開始する時に commandBias からコピーされる。
- ダメージ、ガード、行動CD、調子増減は activeCommandBias を見る。
- 適正距離だけは commandBias を見るため、移動傾向は即変わる。

現在のUI:
- 味方ステータスカードの調子バー下に表示。
- 左が盾、防御寄り。
- 右が剣、攻撃寄り。
- 白く塗られている位置が commandBias。
- 白い枠線の位置が activeCommandBias。
- 現在のコードでは、メーターを直接クリック/ドラッグして変更する処理は無効化されている。
- 現在、調子による攻防メーター自動変化も停止中。
- 変更は指示スキルで行う。

===============================================================================
11. 攻防メーターの効果値
===============================================================================

値 | ガード率 | 行動CD | 調子増加 | 調子減少 | 与ダメ | 被ダメ
---|----------|--------|----------|----------|--------|--------
-2 | x3.0     | x2.5   | x0.6     | x1.5     | x0.8   | x0.8
-1 | x1.8     | x1.6   | x0.8     | x1.2     | x0.9   | x0.9
 0 | x1.0     | x1.0   | x1.0     | x1.0     | x1.0   | x1.0
+1 | x0.6     | x0.8   | x1.0     | x1.0     | x1.1   | x1.15
+2 | x0       | x0.6   | x1.1     | x0.9     | x1.2   | x1.3

行動CD:
- 基準行動CDは6秒。
- 実CD = 6秒 * 攻防メーターactionCd / (1 + 行動速度補正)
- 防御寄りは遅くなる。
- 攻撃寄りは速くなる。

ガード:
- ガード率 = 実ガード率 * 攻防メーターguard倍率。
- +2はガード率0なのでガードできない。
- ガード率が100%を超えた分は、ガード時の軽減率にも一部加算される。

ダメージ:
- 与ダメ/被ダメ補正は、他のダメージ率補正と加算的に合算される。
- 例: 攻防+2は与ダメ+20%、被ダメ+30%相当。
- 例: 攻防-2は与ダメ-20%、被ダメ-20%相当。

調子:
- 調子増加/減少の両方に攻防補正がかかる。
- 防御寄りは調子が上がりにくく、下がりやすい。
- 攻撃+2は調子が少し上がりやすく、下がりにくい。

12. 攻防メーターによる適正距離
===============================================================================

攻防メーターは味方AIの基本適正距離にも影響する。

値 | 前衛距離 | 後衛距離
---|----------|--------
-2 | 160px    | 270px
-1 | 100px    | 250px
 0 | 65px     | 230px
+1 | 50px     | 210px
+2 | 40px     | 200px

前衛扱いの武器:
- 片手剣
- 両手剣
- 拳具
- 棒具

後衛扱いの武器:
- 杖
- 魔導書
- 楽器

武器が判定できない場合:
- role が mage/caster/support なら後衛。
- それ以外は前衛。

この距離にさらに調子による距離倍率がかかる。

13. 指示スキルによる攻防変更
===============================================================================

防御指示:
- 対象1人の攻防メーターを-1。
- CD 7.5秒。
- 射程 480px。

攻撃指示:
- 対象1人の攻防メーターを+1。
- CD 7.5秒。
- 射程 480px。

防御陣形:
- 範囲内の味方全員の攻防メーターを-1。
- CD 20秒。
- 半径 520px。

攻撃陣形:
- 範囲内の味方全員の攻防メーターを+1。
- CD 20秒。
- 半径 520px。

共通:
- 値は必ず-2から+2に丸められる。
- 指示が成功すると支援系の必殺ゲージ獲得対象になる。
- 全体指示は味方ごとに個別に成功/無視判定を行う。

指示無視:
- 攻撃寄り指示(delta > 0): 対象の調子が30以下だと無視される可能性がある。
- 防御寄り指示(delta < 0): 対象の調子が70以上だと無視される可能性がある。
- 無視率 = 75% - スキル強化Lv * 10%
- 無視されると攻防メーターは変わらず、「無視」と表示される。

14. 調子による攻防メーター自動変化
===============================================================================

現在、この仕様は一旦停止中。
味方CPは行動開始時に調子で攻防メーターを自動変更しない。
攻防メーターは指示スキルなど、明示的な変更でのみ動く。

停止前にあった仕様メモ:
- 各CPの最初の5回の行動では自動変化しない。
- 6回目以降の行動開始時から判定する。
- 手動/指示による変更はこの猶予中でも有効。

停止前の低調子判定:
- 調子20以下:
  - 10%で攻防メーターを-2にする。
  - 追加の25%で現在値から-1する。
  - 上記は同じ乱数の範囲判定なので、合計35%で防御寄りに動く。
- 調子40以下:
  - 15%で現在値から-1する。

停止前の高調子判定:
- 調子80超え:
  - 10%で攻防メーターを+2にする。
  - 追加の25%で現在値から+1する。
  - 上記は同じ乱数の範囲判定なので、合計35%で攻撃寄りに動く。
- 調子65以上:
  - 15%で現在値から+1する。

判定順:
- 調子20以下の判定が最優先。
- 次に調子40以下。
- 次に調子80超え。
- 最後に調子65以上。

===============================================================================
15. 注意点・今後確認したいこと
===============================================================================

現在の仕様上、攻防メーターは「表示値」と「実適用値」がずれる時間がある。
これは意図的な仕様で、指示を受けても次の行動開始までは戦闘補正が切り替わらない。

一方で適正距離だけは表示値を即参照する。
そのため、指示直後に立ち位置だけ先に変わり、ダメージやガードは次行動から変わる。

古いメモには攻防メーターの直接クリック/ドラッグ操作があるが、現在のコードでは無効。
必要なら今後、UI操作でcommandBiasを直接変える仕様として復活させる。

調子の増減は攻防メーター、無被弾時間、90以上減衰が絡むため、見た目より増減量が変わる。
特に高調子時は、増加量0.5倍と被ダメ増加があるため、上がりきった後はリスクが大きい。
